Simple briefing

Docker is like a sealed mini Linux computer inside your Windows PC. UNATRARE uses the Pear runtime, which does not run natively on Windows, so Docker gives it the Linux environment it expects.

  • Docker Desktop runs the container.
  • The container runs the UNATRARE node.
  • Docker volumes save your node identity and archive cache between restarts.
  • Your PC must stay awake for heartbeats to keep counting.

Is this safe?

It is reasonably safe for a spare or always-on machine, but it is still third-party open-source P2P software. Treat it like running a torrent-style network node.

  • It should only need public wallet addresses.
  • Never enter a seed phrase, private key, or wallet password.
  • Your IP can be visible to peers. Use a VPN if that bothers you.
  • No router port forwarding is required by the official Docker setup.
  • Recommended space: 10GB minimum, 50GB for long-term comfort.

Windows Docker quickstart

Open PowerShell or Windows Terminal. Copy each block using the button, paste, and press Enter.

Install Docker Desktop first

Download Docker Desktop from the official Docker website. During setup, accept WSL2/Hyper-V defaults, restart if asked, then open Docker Desktop and wait until it says the engine is running.

Official download
https://www.docker.com/get-started/
Install Git if Windows does not have it

If the next step says git is not recognized, install Git from the official source.

Official download
https://git-scm.com/download/win
Clone the official UNATRARE node repo
PowerShell install line
git clone --branch unatrare/phase-0 https://github.com/joeatang/unatrare-intercom "$env:USERPROFILE\unatrare-node"; cd "$env:USERPROFILE\unatrare-node"; copy .env.example .env
Add your public addresses

This opens the config file. Replace the placeholder values with your public addresses. BTC address is required. XCP and TAP are optional, but useful for display/future reward logic.

Open config
notepad .env
.env example
BTC_ADDRESS=your_bitcoin_address
XCP_ADDRESS=your_counterparty_address
TAP_ADDRESS=your_tap_address
Start the node
Start command
docker compose up -d
Watch the logs

Look for a line like [unatrare] Node registered.... That means the node is alive.

Log command
docker compose logs -f
Frog wisdom: keep Docker Desktop running and stop Windows from sleeping if you want steady heartbeats.

Stop the node

This pauses heartbeats but keeps your saved Docker volumes.

PowerShell
cd "$env:USERPROFILE\unatrare-node"; docker compose down

Start again later

Use this after rebooting or after Docker Desktop is running again.

PowerShell
cd "$env:USERPROFILE\unatrare-node"; docker compose up -d

Update the node

Pull the latest code, rebuild the Docker image, and restart.

PowerShell
cd "$env:USERPROFILE\unatrare-node"; git pull; docker compose build --no-cache; docker compose up -d

Do not feed the frog your seed phrase

The node registration uses public addresses as identity/reward markers. It should never need a private key, recovery phrase, wallet file, signing approval, or password. If anything asks for those, stop immediately and verify the source.
